# Memory Operations

Use this file when the user wants better long-term memory behavior rather than just prettier notes.

## Split memory by time horizon

### `memory/YYYY-MM-DD.md`

Use for:

- raw logs
- quick captures
- partial thoughts
- notes from conversations
- temporary reminders

Do not over-edit these files. They are scratchpads and evidence.

### `MEMORY.md`

Use for:

- stable preferences
- durable facts about the human
- repeated workflows
- important decisions that remain relevant
- lessons learned worth preserving

Do not use it for:

- full transcripts
- short-lived task chatter
- project details that belong in a project note
- sensitive information unless intentionally stored

## Distillation workflow

1. Review the last few daily logs
2. Extract durable facts, decisions, and lessons
3. Merge duplicates into a single clean memory entry
4. Remove stale or contradicted items
5. Link to supporting project or concept notes when useful

## Good memory entry examples

Good:

- User prefers short direct replies in chat, fuller explanations only for planning
- Project X uses Obsidian as the source of truth for durable notes
- Weekly review should happen on Sunday evening

Bad:

- Today we talked for 40 minutes about several random things
- The user said hello and then asked about a plugin
- Full bullet-by-bullet meeting transcript

## Suggested sections for `MEMORY.md`

```markdown
# Long-Term Memory

```

## Privacy rule

If a workspace is shared or exposed to group contexts, be careful about what goes into long-term memory. Favor project notes or scoped files over broad personal memory when needed.
